# Markdown 
- é uma linguagem de marcação facil de usar, onde o foco é formatar textos de uma maneira mais "rapida e eficiente", com o foco em web.
- Com o makdown é possivel adicinar imagens, listas, links, títulos, Negrito, Italico. Tudo isso com uma formatação básica e usando uma sintaxe leve e intuitiva.
- é uma ferramenta usada para quem quer usar textos com uma formataçao limpa, e deixar uma boa imagem no seu texto.

# Sixtaxe basica do markdown

- #Titulo 
- ##Subtitulo 
- ###sub-subtitulo
- **negrito** ou __negrito__
- *itálico* ou _itálico_


# Paradigmas de Programação
- é um comglomerado de regras, conceito de design e princípios que define como o código do programa vai ser escrito e organizado.
-Cada paradigma tem a sua propria escrita e organização
-Fazendo com que o desenvolvedor possa ter mais liberdade para conceituar e resolver problemas mais complexos, por meio de uma linguiagem de programaçào


# Abordagens
## Paradigma Imperativo: 
- Paradigma Imperativo é uma programação "Passo a Passo"
Ou seja, um programa do estilo imperativo, ele precisa de um conjunto de instruções, e cada intrucão tem que ser declarada e orientada de como o programa deve ser feito.
## Paradigma Declarativo
- O paradigma declarativo é uma abordagem de programação focada no "que" deve ser feito, em vez de "como" fazer, totalmente ao contrario do imperativo.
- ele descreve o resultado desejado, delegando ao sistema a lógica de execução passo a passo, que resulta em maior concisão, abastração e legibilidade.
- **Abstração** é onde esconde a complexibilidade dos algoritmos subjacentes. 

### Exemplos de Imperativo
- Programação estruturada
- Programação Procedual
- Programação Orientada a Objeto (POO) 

# Estruturada 
- A programação estruturada ela é referida como "Programação modular"
- Ele serve para otimizar o código utilizando os blocos de fluxo de controle de programa, bloco de tomadas de decisão (if, if then, else) E as contruções de iteração (For, While loops) e funções.
- A principal função é melhorar a estrutura e a estabilidade do código. Facilitando a reutilização do código.

- Estruturada sáo Cobol, Php, pearl e Go.

# Procedural
- É uma programação baseado no conceito de chamadas de procedimento, conhecidas como (sub-rotinas ou funções)
- Eles são uma lista de instruções para dizer passo a passo o que o computador deve fazer.
Procedurais: C++, C, Java e Pascal. 

### Exemplo:
- ### **Paradigma:** 
 Ir para a cozinha; Obtenha o Açucar e o café, Mistura os dois e aqueça no fogo, depois de ferver pegue uma caneca e coloca a bebida no recipiente e beba;

 ### **Declarativo:**
 - cozinha, açucar, café, quente;

# Programação Orientada A Objeto (POO)
- Criado em 1970, O POO, se baseia na criação de objetos que possuem propriedades e métodos, permitindo o encapsulamento de dados e a reutilização do código. Em poo, todos os elementos são metodos.

- Linguagens: PHP, Java, c#, JS, Python e Ruby.

## Pilares
- **Herança**
- **Polimorfismo**
- **Encapsulamento**
- **Abstração**

### Benefícios 
- **Reutilização de código**
- **Flexibilidade e escalabilidade**
- **Modularidade**
- **Organização e coesão do código**

# *Conclusão*
- ## **Paradigmas Imperativos:** 
Foco em **Como** executar, define o fluxo de controle que altera o estado do programa.

### exemplos:
- Procedural, Estruturada e POO

- ## **Paradigmas Declaraativos:**
Foco em **o que** executar, define a lógica do programa, mas náo em um fluxo de controle detalhado.

### exemplos:
- Funcional e Lógico.
